# mem-tool

macOS 系统内存监控 TUI 工具，实时追踪进程内存使用，识别内存占用高或增长异常的进程。

## 核心职责

1. **进程采集** — 定期采集系统上所有进程的内存使用（排除系统进程），按内存降序展示 Top N
2. **增长率追踪** — 滑动窗口内计算每个进程的内存增长趋势
3. **阈值告警** — 单进程超限、增长异常、系统内存超限时，发送 macOS 通知（带冷却）
4. **CSV 日志** — 每轮采集结果追加写入 CSV，供离线分析
5. **TUI 展示** — 终端内原地刷新的仪表盘，显示进程列表、状态指示、告警历史

## 不做的事

- 不管理/控制进程（不 kill、不 restart）
- 不做跨平台支持（仅 macOS，依赖 `footprint`、`vm_stat`、`sysctl`、`osascript`）
- 不提供 Web UI 或远程访问
- 不做历史数据可视化（CSV 导出后用其他工具分析）

## 采集策略

1. `ps aux` 单次调用获取全量 RSS + VSZ
2. 排除 root (UID 0) 系统进程
3. 按 RSS 降序排序
4. 仅对 Top 20 调用 `footprint` 获取精确物理内存

### 展示指标

| 指标 | 来源 | 用途 |
|------|------|------|
| **footprint** | `footprint <pid>`，Top 20 | 真实物理内存，用于增长率计算和告警 |
| **VSZ** | `ps aux`，全量 | 虚拟内存大小，用于发现异常膨胀进程 |
| **RSS** | `ps aux`，全量 | 物理驻留内存，用于粗排序 |

### 默认阈值

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 采集间隔 | 30s | |
| 单进程上限 | 1500MB | footprint |
| 增长窗口 | 30m | |
| 增长上限 | 200MB/30m | footprint |
| 系统内存上限 | 150GB | VSZ 维度，CLI 参数可配置 |
| 告警冷却 | 5m | |
| Top N | 20 | footprint 精确采集数量 |
| CSV 路径 | ~/mem-tool-log.csv | |
